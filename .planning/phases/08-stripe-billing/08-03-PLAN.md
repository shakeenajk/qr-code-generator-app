  <name>Task 1: Create checkout session endpoint and subscription status endpoint</name>
  <files>src/pages/api/checkout/create.ts, src/pages/api/subscription/status.ts</files>
  <action>
Create `src/pages/api/checkout/create.ts` following RESEARCH.md Pattern 3 exactly:
- `export const prerender = false`
- `POST: APIRoute` — extract `userId` from `locals.auth()`, return 401 if absent
- Parse `{ priceId }` from `request.json()`
- Check DB for existing `stripeCustomerId` using `db.query.subscriptions.findFirst({ where: eq(subscriptions.userId, userId) })`
- If no stripeCustomerId: use `@clerk/astro/server` `clerkClient` to get user email, create Stripe customer with `metadata: { clerkUserId: userId }`
- Call `stripe.checkout.sessions.create({ customer: customerId, mode: 'subscription', line_items: [{ price: priceId, quantity: 1 }], success_url: `${import.meta.env.PUBLIC_BASE_URL}/dashboard?upgraded=true`, cancel_url: `${import.meta.env.PUBLIC_BASE_URL}/pricing`, metadata: { clerkUserId: userId }, subscription_data: { metadata: { clerkUserId: userId } } })`
- Return `{ url: session.url }` with status 200
- Wrap Stripe API call in try/catch, return 500 with `{ error: 'Failed to create checkout session' }` on failure

IMPORTANT anti-pattern to avoid: Do NOT create a new Stripe customer if `stripeCustomerId` already exists in DB. Always pass the existing `customer` ID to avoid duplicate customers.

Create `src/pages/api/subscription/status.ts` following RESEARCH.md code example exactly:
- `export const prerender = false`
- `GET: APIRoute` — extract `userId` from `locals.auth()`, return 401 if absent
- Query DB for subscription row
- Return `{ tier: sub?.tier ?? 'free', status: sub?.status ?? 'inactive', paymentFailed: sub?.status === 'past_due' }`
- Content-Type: application/json
