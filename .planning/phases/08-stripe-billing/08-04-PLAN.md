Create `src/components/billing/` directory and 4 component files.

**UpgradeCTAPanel.astro:**
Props: `{ tier: 'free' | 'starter' | 'pro' }`
- If tier === 'free': render a persistent upgrade panel at the bottom of sidebar. Style: rounded-lg border border-blue-200 dark:border-blue-800 bg-blue-50 dark:bg-blue-900/20 p-4 mx-3 mb-3. Content: "Upgrade to Pro" heading (text-sm font-semibold text-blue-700 dark:text-blue-400), one-line description "Unlock dynamic QR codes and analytics", CTA button that posts to /api/checkout/create with STRIPE_PRICE_PRO_MONTHLY and redirects to Stripe. Button style: w-full bg-[#2563EB] text-white text-xs font-semibold py-2 rounded-lg hover:bg-blue-700 transition-colors.
- If tier === 'starter' or 'pro': render a simple "Manage subscription" link that POSTs to /api/portal/create and redirects to portal URL. Style: text-xs text-gray-500 dark:text-slate-400 hover:text-gray-700 dark:hover:text-slate-200 transition-colors px-4 mb-3 block text-center.
- The CTA button is an `<a>` that calls a JS fetch on click (not a form POST) — use inline onclick script or a small client:load island. Simplest approach: `<button onclick="handleUpgrade()">` with an inline `<script>` that posts to the API and redirects to the returned URL.

**TierBadge.tsx:**
Props: `{ tier: 'free' | 'starter' | 'pro' }`
- If tier === 'free': return null (no badge)
- If tier === 'starter': return `<span className="text-xs font-semibold px-2 py-0.5 rounded-full bg-blue-100 text-blue-700 dark:bg-blue-900/40 dark:text-blue-400 ml-1.5">Starter</span>`
- If tier === 'pro': return `<span className="text-xs font-semibold px-2 py-0.5 rounded-full bg-purple-100 text-purple-700 dark:bg-purple-900/40 dark:text-purple-400 ml-1.5">Pro</span>`
- TierBadge must accept tier as a prop — it does NOT fetch subscription status itself (parent passes tier). This keeps it pure and testable.

**PaymentFailureBanner.astro:**
Props: `{ show: boolean }`
- If show is false: render nothing
- If show: render a dismissible warning banner. Full-width, sticky at top of main content area. Style: bg-red-50 dark:bg-red-900/20 border-b border-red-200 dark:border-red-800 px-6 py-3. Content: warning icon + "Your payment failed — please update your payment method to keep your subscription active." + "Update payment" button that POSTs to /api/portal/create. Text: text-sm text-red-700 dark:text-red-400.

**SubscriptionPolling.tsx:**
Implement exactly as RESEARCH.md Pattern 5. Key requirements:
- Only activates when `window.location.search` contains `upgraded`
- Polls `/api/subscription/status` every 500ms up to 10 attempts (5 seconds total)
- Shows "Activating your plan..." indicator while polling (centered, with a spinner or text, data-testid="activating-indicator")
- On success (tier !== 'free'): toast "Welcome to [Starter/Pro]! Your plan is now active." — use a simple fixed-position toast div since no toast library is established; auto-dismiss after 5s
- On timeout (still 'free' after 10 attempts): silently dismiss, remove ?upgraded param
- `window.history.replaceState({}, '', '/dashboard')` to clean URL after polling ends
- Export default, no props needed
