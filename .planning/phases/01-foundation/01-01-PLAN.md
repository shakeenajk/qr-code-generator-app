<task type="auto">
  <name>Task 1: Scaffold Astro project and install all dependencies</name>
  <files>package.json, astro.config.mjs, tsconfig.json, src/styles/global.css</files>
  <action>
Run the following commands in sequence from the repo root (`/Users/ranjit/Documents/Development/qr-generator`):

```bash
npm create astro@latest . -- --template minimal --no-install --no-git
npm install
npx astro add react --yes
npm install tailwindcss @tailwindcss/vite
npx astro add sitemap --yes
```

After installation, open `astro.config.mjs` and ensure it matches this exact structure:

```javascript
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';
import react from '@astrojs/react';
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://qrcraft.app',
  integrations: [react(), sitemap()],
  vite: {
    plugins: [tailwindcss()],
  },
});
```

CRITICAL: `site: 'https://qrcraft.app'` MUST be present. Without it, `@astrojs/sitemap` produces no output and `Astro.site` is `undefined` in layouts.

Do NOT use `@astrojs/tailwind` — it is deprecated for Tailwind v4. The `@tailwindcss/vite` plugin in `vite.plugins` is the correct integration.

Create `src/styles/global.css` with:

```css
@import "tailwindcss";

@layer base {
  :root {
    --color-brand: #2563EB;
  }
}
```

Verify `tsconfig.json` has `"jsx": "react-jsx"` in compilerOptions (added by `npx astro add react`).

<task type="auto">
  <name>Task 2: Create static robots.txt and install Playwright</name>
  <files>public/robots.txt, playwright.config.ts, package.json</files>
  <action>
Create `public/robots.txt` with this exact content (static file served directly by Astro from the public directory):

```
User-agent: *
Allow: /

Sitemap: https://qrcraft.app/sitemap-index.xml
```

Install Playwright and initialize its config:

```bash
npm init playwright@latest -- --quiet --browser=chromium --browser=firefox --browser=webkit --no-examples --lang=ts
```

After initialization, open `playwright.config.ts` and update the `webServer` block and `baseURL`:

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: 'list',
  use: {
    baseURL: 'http://localhost:4321',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit', use: { ...devices['Desktop Safari'] } },
  ],
  webServer: {
    command: 'npm run preview -- --port 4321',
    port: 4321,
    reuseExistingServer: !process.env.CI,
    timeout: 30000,
  },
});
```

Note: `npm run preview` requires a prior `npm run build`. During development, tasks that run Playwright tests must build first. Add these scripts to `package.json`:

```json
"test": "playwright test",
"test:smoke": "playwright test --grep @smoke"
```

Create an empty `tests/` directory placeholder:

```bash
mkdir -p tests
```
