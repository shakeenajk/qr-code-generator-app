**Static inline SVG thumbnails (per research recommendation — no live QR instances):**

Use 24x24 inline SVGs representing each shape visually. Keep them simple:

DOT_SHAPES array (six entries, type + icon):
- `square`: 3x3 grid of small filled squares (`<rect>` elements)
- `dots`: 3x3 grid of small filled circles (`<circle>` elements)
- `rounded`: 3x3 grid of rounded-corner squares (`<rect rx="2">`)
- `extra-rounded`: 3x3 grid of heavily rounded squares (`<rect rx="4">`)
- `classy`: angled/classy pattern — 3x3 grid with one corner cut (use `<polygon>` or slightly irregular shape)
- `classy-rounded`: same but with rounded corners

For classy/classy-rounded, a simple approach: show a 3x3 dot grid where the top-right dot is replaced by a diagonal corner shape (a right-triangle `<polygon>`) — this communicates the classy "corner" aesthetic. Exact visual is discretionary per CONTEXT.md.

CORNER_FRAMES array (three entries):
- `square`: a square outline (`<rect fill="none" stroke="currentColor">`)
- `extra-rounded`: rounded square outline (`<rect fill="none" stroke="currentColor" rx="4">`)
- `dot`: circle outline (`<circle fill="none" stroke="currentColor">`)

CORNER_PUPILS array (three entries):
- `square`: filled square (`<rect>`)
- `dot`: filled circle (`<circle>`)
- `extra-rounded`: filled rounded square (`<rect rx="3">`)

All SVGs should use `currentColor` fill/stroke so they respond to CSS color. Size each SVG at `width="20" height="20"` viewBox="0 0 20 20"`.

Implementation note: The `cornerSquareType` and `cornerDotType` prop names in ShapeSectionState map to `cornersSquareOptions.type` and `cornersDotOptions.type` in the qr-code-styling `.update()` call respectively — this mapping happens in QRGeneratorIsland (Plan 05), not here.
