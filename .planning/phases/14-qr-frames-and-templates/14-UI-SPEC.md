## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), thumbnail label text gap |
| sm | 8px | Tile internal padding (`p-2`), gap between thumbnail tiles (`gap-2`) |
| md | 16px | Section stacking (`space-y-4`), CTA text input margin-top |
| lg | 24px | Panel section padding (`p-6`), gap between frame picker rows |
| xl | 32px | Customization panel section separator |
| 2xl | 48px | Major panel breaks |
| 3xl | 64px | Page-level spacing (unchanged from Phase 12) |

**Source:** Carried forward from Phase 12 UI-SPEC. `ShapeSection.tsx` uses `gap-2` for tile grids, `mb-4` between subsections, `p-2` for tile internal padding — this phase extends the same pattern.

Exceptions:
- Frame thumbnail tiles: `p-2` (8px) internal padding — matches ShapeSection tile size exactly
- Template preset cards: `p-3` (12px) internal padding to accommodate two-line label under the thumbnail image
- Preview canvas container: no padding change; frame is drawn inside the existing QR preview boundary

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 400 (regular) | 1.4 |
| Heading (section) | 30px (`text-3xl`) | 700 (bold) | 1.2 |
| Display (hero/pricing h1) | 36–48px (`text-4xl` / `text-5xl`) | 700 (bold) | 1.1 |

**Weights declared:** 400 (regular) and 700 (bold) only.

**Phase 14 specific typography:**
- Frame picker subsection label (e.g. "Frame Style"): `text-sm font-semibold text-gray-900 dark:text-white` — matches the existing ShapeSection `h3` pattern (`text-sm font-semibold`)
- Frame category sub-label (e.g. "Borders", "Shapes"): `text-xs text-gray-500 dark:text-slate-400 mb-2` — matches ShapeSection's "Dot style" / "Corner frame" label pattern exactly
- CTA text input label: `block text-sm font-normal text-gray-700 dark:text-slate-300` — matches VCardTab field label pattern
- Template picker section heading: `text-sm font-semibold text-gray-900 dark:text-white mb-3`
- Template preset card name: `text-xs font-normal text-gray-600 dark:text-slate-400 text-center mt-1 leading-tight` — two-line max, truncated with `truncate`
- SVG export notice (toast or inline): `text-sm` at 400 weight

**Note on `font-semibold`:** The ShapeSection `h3` already uses `font-semibold` in production. This phase matches that existing pattern for subsection headings within the customization panel. No new weight class is introduced — `font-semibold` is already in the codebase.

## Component Inventory

These are the specific UI elements introduced or modified in Phase 14.

### Template Picker (`TemplateSection.tsx`)

**Placement:** TOP of the customization panel, before all individual sections (Frame, Color & Shape, Logo). Established by decision D-07.

**Layout:** Horizontal scroll row of preset cards at `md` breakpoint and above; wraps to 2-column grid on mobile. Each card is 80px wide × 96px tall.

**Card anatomy:**
- Thumbnail image: 80×72px, rounded corners `rounded-md`, shows a miniature QR code with the preset frame + color applied (static PNG generated at build time or inline SVG illustration)
- Name label: `text-xs` below thumbnail, centered, max 2 lines
- Selected state: `ring-2 ring-blue-500 ring-offset-1` outer ring
- Default state: `border border-gray-200 dark:border-slate-600 rounded-lg`
- Hover state: `hover:border-gray-400 dark:hover:border-slate-400 transition-colors`

**Categories (4 groups):**
1. Minimal — clean borders, neutral tones
2. Bold — high-contrast fills, strong CTA text
3. Business — professional rectangular frames with bottom banner
4. Vibrant — colorful frames, rounded or badge shapes

**Number of templates:** 16 presets (4 per category). Source: CONTEXT.md D-09 discretion + SUMMARY.md "12-20 curated presets" guidance — 16 is the midpoint, organized evenly across 4 categories.

**One-click apply:** Clicking a preset card immediately applies frame type + foreground color + dot shape to the generator state. No confirmation step. Frame CTA text resets to the frame's default text.

**"No template" option:** First card in the Minimal category is labeled "None" with an empty/plain QR thumbnail — allows user to return to baseline.

### Frame Picker (`FrameSection.tsx`)

**Placement:** After TemplateSection, before ColorSection and ShapeSection in the customization panel.

**Layout:** Grid of clickable thumbnail tiles, `grid-cols-4 gap-2` — matches ShapeSection dot shape grid pattern. 8–10 frame tiles total.

**Tile anatomy (matches ShapeSection tile pattern exactly):**
- Size: `p-2 rounded-lg border-2 flex items-center justify-center`
- Thumbnail: 32×32px inline SVG silhouette of frame shape, `currentColor` fill/stroke
- Default: `border-gray-200 bg-white hover:border-gray-300 text-gray-600 dark:border-slate-600 dark:bg-slate-700 dark:hover:border-slate-500 dark:text-slate-300`
- Selected: `border-blue-600 bg-blue-50 text-blue-600 dark:border-blue-500 dark:bg-blue-900/20 dark:text-blue-400`
- `aria-pressed` attribute for accessibility

**Frame designs (8 total):**

| `simple-border` | Simple Border | Thin rectangular border + bottom text band |

| `top-bottom` | Top & Bottom | Text bands above and below QR |

| `badge` | Badge | Circular badge shape around QR |
| `shopping-bag` | Shopping Bag | Shopping bag silhouette frame |
| `clipboard` | Clipboard | Clipboard silhouette frame |

**Source:** D-01 (8–10 frames covering key categories). "Phone mockup" explicitly excluded per REQUIREMENTS.md Out of Scope.

### CTA Text Input (within FrameSection)

**Visibility:** Shown only when `frameType !== 'none'`. Hidden when "No Frame" is selected (D-06).

**Layout:** Appears directly below the frame tile grid, with `mt-4` spacing.

**Label:** `block text-sm font-normal text-gray-700 dark:text-slate-300 mb-1` — text "CTA Text"

**Input:** `w-full px-3 py-2 border border-gray-300 rounded-md shadow-sm text-sm focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 dark:bg-slate-800 dark:text-slate-100 dark:border-slate-600 dark:placeholder-slate-400` — matches VCardTab input pattern exactly.

**Default values per frame (D-04):**

| Frame ID | Default CTA Text |
|----------|-----------------|
| `simple-border` | "Scan Me" |
| `rounded-border` | "Scan Me" |
| `top-bottom` | "Scan Me" |
| `bottom-banner` | "Scan Here" |
| `badge` | "Scan Me" |
| `shopping-bag` | "Order Here" |
| `clipboard` | "See Menu" |

**Character limit:** 30 characters. Enforced with `maxLength={30}`. Counter shown inline: `text-xs text-gray-400 text-right mt-1` displaying `{n}/30`.

**Frame color:** Follows QR foreground color automatically (D-10). No separate color picker rendered for the frame.

### SVG Export — Frame Active State

**Decision (D-13, Claude's discretion):** Disable the SVG button with a tooltip when a frame is active. Rationale: a toast is ephemeral and users may miss it when quickly switching export formats; a disabled state with tooltip makes the constraint permanently visible and prevents repeated failed attempts.

**Implementation:**
- SVG button receives `disabled` attribute when `frameType !== 'none'`
- Button visual: `opacity-50 cursor-not-allowed` (existing disabled pattern)
- Tooltip text shown on hover via `title` attribute: `"SVG export is frameless. Use PNG to include the frame."`
- PNG button remains fully active and is the recommended path

**Source:** D-13 discretion. REQUIREMENTS.md Out of Scope confirms: "SVG export with frames — Canvas composition is raster-only; SVG export stays frameless."

---

### Export Buttons — PNG Composition Branch

**Modified behavior in `ExportButtons.tsx`:** When `frameType !== 'none'`, PNG download and Copy PNG paths call `composeQRWithFrame(qrBitmap, frameConfig)` before triggering download/copy. No visual change to the button itself — user sees the same "Download PNG" button.

**Loading state during composition:** PNG button shows a brief `disabled` + spinner state while Canvas composition runs (Canvas 2D is synchronous but `toBlob()` is async). Duration expected < 100ms; loading indicator optional but recommended for perceived polish.

## Interaction States

### Frame thumbnail tile
- Default: `border-2 border-gray-200 bg-white text-gray-600 dark:border-slate-600 dark:bg-slate-700 dark:text-slate-300`
- Hover: `border-gray-300 dark:border-slate-500`
- Selected (active): `border-blue-600 bg-blue-50 text-blue-600 dark:border-blue-500 dark:bg-blue-900/20 dark:text-blue-400`
- Focus-visible: `ring-2 ring-blue-500 ring-offset-2` (keyboard navigation)
- `aria-pressed={true}` when active

### Template preset card
- Default: `border border-gray-200 dark:border-slate-600 rounded-lg`
- Hover: `border-gray-400 dark:border-slate-400 transition-colors`
- Selected: `ring-2 ring-blue-500 ring-offset-1 border-transparent`
- Focus-visible: `ring-2 ring-blue-500 ring-offset-2`

### CTA text input
- Default: `border-gray-300 dark:border-slate-600`
- Focus: `ring-2 ring-blue-500 border-blue-500`
- Filled (has text): no visual change from default
- Over limit (> 30 chars): prevented by `maxLength`, no visual error needed

### SVG export button (frame active)
- Disabled: `opacity-50 cursor-not-allowed`
- Tooltip on hover: visible via native `title` attribute

### PNG export button (composing)
- Composing: `disabled opacity-75` + spinner icon (16px `animate-spin` Lucide `Loader2`)
- Complete: returns to default active state
