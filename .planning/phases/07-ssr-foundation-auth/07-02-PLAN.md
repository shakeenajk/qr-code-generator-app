<task type="auto">
  <name>Task 2: Wire Clerk middleware for /dashboard protection</name>
  <files>src/middleware.ts</files>
  <action>
    Create `src/middleware.ts` with Clerk middleware that protects `/dashboard` and all sub-routes.

    ```typescript
    // src/middleware.ts
    import { clerkMiddleware, createRouteMatcher } from '@clerk/astro/server';

    const isProtectedRoute = createRouteMatcher(['/dashboard(.*)']);

    export const onRequest = clerkMiddleware((auth, context) => {
      const { redirectToSignIn, userId } = auth();
      if (!userId && isProtectedRoute(context.request)) {
        return redirectToSignIn();
      }
    });
    ```

    This middleware:
    - Runs on every request (Astro middleware convention: export `onRequest`)
    - Uses `createRouteMatcher` to match `/dashboard` and any sub-path (e.g., `/dashboard/analytics`)
    - Redirects unauthenticated users to Clerk's sign-in page (which will be `/login` after the Clerk Dashboard sign-in URL is configured)
    - Does nothing for public routes (homepage, /login, /signup, etc.)

    The middleware lives in `src/middleware.ts` (not `/middleware.ts` at root). Astro picks it up automatically. Do NOT place it in `api/` or `pages/`.

    Note: This middleware is in serverless mode because we did not enable edgeMiddleware in the Vercel adapter. This is correct and intentional — Clerk's middleware is incompatible with Vercel Edge middleware.
