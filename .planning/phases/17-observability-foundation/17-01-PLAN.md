  <action>
1. Install the Sentry Astro SDK:
   ```
   npx astro add @sentry/astro
   ```
   If the interactive prompt fails, install manually:
   ```
   npm install @sentry/astro
   ```

2. The `npx astro add` command may auto-create `sentry.client.config.ts` and `sentry.server.config.ts` in the project root. If it does NOT create them, create them manually:

   **sentry.client.config.ts** (project root):
   ```typescript
   import * as Sentry from "@sentry/astro";

   Sentry.init({
     dsn: import.meta.env.PUBLIC_SENTRY_DSN,
     integrations: [Sentry.browserTracingIntegration()],
     tracesSampleRate: 0.1,
     replaysSessionSampleRate: 0,
     replaysOnErrorSampleRate: 0,
   });
   ```

   **sentry.server.config.ts** (project root):
   ```typescript
   import * as Sentry from "@sentry/astro";

   Sentry.init({
     dsn: import.meta.env.PUBLIC_SENTRY_DSN,
     tracesSampleRate: 0.1,
   });
   ```

3. Update `astro.config.mjs` to add the Sentry integration (if `npx astro add` did not already). The integration MUST include `sourceMapsUploadOptions` with `release.name` set to `process.env.VERCEL_GIT_COMMIT_SHA` for source maps to work on Vercel. Add the sentry integration to the integrations array:
   ```javascript
   import sentry from '@sentry/astro';

   // In the integrations array, add:
   sentry({
     sourceMapsUploadOptions: {
       project: 'qrcraft',
       authToken: process.env.SENTRY_AUTH_TOKEN,
       release: {
         name: process.env.VERCEL_GIT_COMMIT_SHA ?? 'local-dev',
       },
     },
   }),
   ```
   IMPORTANT: The `release.name` MUST use `process.env.VERCEL_GIT_COMMIT_SHA` (not import.meta.env). This is a Vite plugin config running at build time in Node, not in the browser. Without this, source maps upload but are never associated with the deployed version (Pitfall 9 from research).

4. Add these environment variables to `.env.example` (create if needed) and document them:
   - `PUBLIC_SENTRY_DSN` — Sentry project DSN (public, safe for client)
   - `SENTRY_AUTH_TOKEN` — Sentry auth token for source map uploads (secret, build-time only)

5. Do NOT add `.env` to git. Verify `.gitignore` already includes `.env`.

Note: The user will need to create a Sentry project and obtain these values. The code should work gracefully when DSN is not set (Sentry SDK silently no-ops when DSN is undefined).

   import type { APIRoute } from 'astro';

   export const GET: APIRoute = async () => {
     // Only allow in non-production or when explicitly enabled
     if (import.meta.env.PROD && !import.meta.env.SENTRY_DEBUG_ENABLED) {
       return new Response(JSON.stringify({ error: 'Not available' }), {
         status: 404,
         headers: { 'Content-Type': 'application/json' },
       });
     }

     throw new Error('Sentry debug test — this error should appear in Sentry dashboard');
   };
