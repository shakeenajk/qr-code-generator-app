## Spacing Scale

Declared values (must be multiples of 4). Source: inferred from existing dashboard patterns (`p-6 md:p-8`, `mb-8`, `gap-2`, `gap-4`, `py-20`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding (existing: `gap-1`, `px-2 py-1`) |
| sm | 8px | Compact element spacing, button gaps (existing: `gap-2`, `p-2`) |
| md | 16px | Default element spacing, card internal padding (existing: `p-4`, `gap-4`) |
| lg | 24px | Section padding, page content padding (existing: `p-6`) |
| xl | 32px | Page header bottom margin, layout gaps (existing: `p-8`, `mb-8`) |
| 2xl | 48px | Major section breaks between analytics panels |
| 3xl | 64px | Empty state vertical padding (existing: `py-20` ~80px; use 64px for analytics empty) |

Exceptions:
- Stat card grid: `gap-4` (16px) between cards on mobile, `gap-6` (24px) on md+ screens
- Progress bar row: `gap-3` (12px) between label, bar, and count — allowed as a 4px-multiple exception for density
- Touch targets for icon-only buttons (Analytics, Edit, Delete): minimum 36px height via `py-1.5 px-2.5` — matches existing card action buttons

## Typography

Source: inferred from existing `QRLibrary.tsx` and `dashboard/index.astro` patterns. No custom font — Tailwind system stack.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Card metadata, progress bar labels, tooltip copy, country/device names |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.4 | Stat card sub-labels ("Total Scans"), badge text, section sub-headers |
| Heading | 20px (`text-xl`) | 700 (bold) | 1.2 | Stat card numbers (total/unique scan counts), section headings |
| Display | 24px (`text-2xl`) | 700 (bold) | 1.2 | Page heading (QR name — "Analytics: [QR Name]"), matches dashboard `h1` |

Font weight contract: regular (400) for body/metadata, bold (700) for headings/numbers. Semibold (600) reserved for labels and secondary emphasis only (matches existing `font-semibold` badge pattern).

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|

1. Tremor AreaChart fill color (`indigo`) — CONTEXT.md D-06 specifies indigo to match dashboard accent

## Component Inventory

New components required for Phase 11:

### AnalyticsPage (`src/pages/dashboard/analytics/[slug].astro`)
- SSR Astro page (source: CONTEXT.md "Claude's Discretion")
- Wraps `DashboardLayout` with `activeSection="my-qr-codes"` and `tier` prop
- Page header: QR name as `<h1>` (Display/24px/bold) + slug in `text-xs text-gray-500` + back link `← My QR Codes` linking to `/dashboard`
- Content padding: `p-6 md:p-8` — exact match to dashboard/index.astro

### StatCard (inline component or small React component)
- Two-up grid: `grid grid-cols-2 gap-4 md:gap-6`
- Each card: `bg-white dark:bg-slate-800 rounded-xl p-5 border border-gray-200 dark:border-slate-700`
- Label: 12px semibold, `text-gray-500 dark:text-slate-400`
- Number: 20px bold, `text-gray-900 dark:text-white`
- Unique scan card shows label as "~Unique Scans" with tilde prefix to communicate approximation (CONTEXT.md D-16)

### ScanChart (React island, `client:only="react"`)
- Tremor `AreaChart` component
- Data: array of `{ date: string, scans: number }` — 30 entries, one per day
- X-axis: abbreviated date label (`Mar 1`, `Mar 15`, `Mar 30`)
- Y-axis: scan count integer
- Colors: `["indigo"]` — Tremor color token (CONTEXT.md D-06, D-07)
- Dark mode: Tremor built-in dark mode support (CONTEXT.md D-08)
- Tooltip: shows `"date — N scans"` on hover
- Height: `h-64` (256px) on mobile, `h-72` (288px) on md+
- Loading state: `h-64 bg-gray-100 dark:bg-slate-800 rounded-xl animate-pulse`

### ProgressBarList (reusable for device + country sections)
- Section wrapper: `bg-white dark:bg-slate-800 rounded-xl p-5 border border-gray-200 dark:border-slate-700`
- Section heading: 14px semibold, `text-gray-700 dark:text-slate-300 mb-4`
- Each row: label (14px regular) | fill bar | count + percentage (12px regular, right-aligned)
- Row spacing: `space-y-3`
- Bar container: `flex-1 h-2 bg-gray-200 dark:bg-slate-700 rounded-full overflow-hidden`
- Bar fill: `h-2 bg-indigo-500 dark:bg-indigo-400 rounded-full` with inline `style="width: {pct}%"`
- Device labels: "iOS", "Android", "Desktop" (sentence case, not all-caps)
- Country labels: ISO 3166-1 alpha-2 code + country name where known (e.g. "US — United States"), fallback to code only

### AnalyticsButton (addition to `QRLibrary.tsx` dynamic card actions)
- New icon button alongside Edit and Delete in `CardActions`
- Icon: `BarChart2` from lucide-react (or `LineChart` — choose `BarChart2` for semantic clarity)
- Appearance: `flex items-center gap-1 text-xs px-2.5 py-1.5 rounded-md border border-indigo-200 dark:border-indigo-900 text-indigo-600 dark:text-indigo-400 hover:bg-indigo-50 dark:hover:bg-indigo-950 transition-colors`
- Label: "Analytics" (text visible, not icon-only — matches existing Edit/Delete button pattern)
- Navigates to `/dashboard/analytics/[slug]` — standard `<a href>` link, no JS navigation needed
- Only rendered when `qr.isDynamic === true` and `qr.slug !== null`

## States and Interactions

### Loading State
- Page loads server-side (SSR) for totals, device, countries — no loading skeleton needed for these
- ScanChart loads client-side as React island — show `h-64 rounded-xl bg-gray-100 dark:bg-slate-800 animate-pulse` until chart renders
- No full-page loading spinner — partial island loading is acceptable

### Empty State (0 scans — CONTEXT.md "Claude's Discretion")
- Stat cards show `0` for both Total and ~Unique
- Chart panel shows empty state message instead of chart: centered text block
  - Heading (14px semibold): "No scans yet"
  - Body (14px regular, gray): "Share your QR code to start tracking."
  - No illustration needed — minimal text-only empty state consistent with panel design
- Device and country panels show: "(No data)" row in gray, 12px regular
- No "Go to Generator" CTA — user is in analytics context, not library

### Error State
- API fetch failure (non-200 from `/api/analytics/[slug]`):
  - Replace page content with centered error panel: `text-red-600 dark:text-red-400`
  - Heading: "Could not load analytics"
  - Body: "Something went wrong fetching your scan data. Try refreshing the page."
  - Retry link: "Refresh" — `<a href={window.location.href}>` — text-indigo-600
- 403 (non-Pro user): redirect to `/pricing` with Astro server-side redirect in the `.astro` page frontmatter
- 404 (slug not owned by user): redirect to `/dashboard` with 302

### Analytics Button on Library Card
- Button is an anchor (`<a>`) not a `<button>` — avoids JS navigation, allows browser back
- No loading state on the button itself — navigation is near-instant
- No confirmation required — analytics is read-only, non-destructive

## Accessibility

- Back link uses `aria-label="Back to My QR Codes"` in addition to visible text
- AreaChart wrapper: `aria-label="Scan volume over last 30 days"` on the container div
- Progress bars: `role="progressbar" aria-valuenow={pct} aria-valuemin={0} aria-valuemax={100} aria-label="{device or country name}: {pct}%"`
- Stat card numbers: wrapped in `<p aria-live="polite">` if loaded client-side
- Analytics button on card: `aria-label="View analytics for {qr.name}"`
- Color is never the sole differentiator — each bar has a text percentage label
