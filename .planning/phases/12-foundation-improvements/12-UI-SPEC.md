## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Label-to-input gap (`space-y-2`), button internal vertical padding (`py-2`) |
| md | 16px | Form field stacking (`space-y-4` = 16px), section horizontal padding (`px-4`) |
| lg | 24px | Card internal padding (`p-6` = 24px), FAQ item padding |
| xl | 32px | Section vertical rhythm (`py-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level section spacing (`py-16`) |

**Source:** Detected from existing Tailwind utility usage in `Header.astro`, `VCardTab.tsx`, `FAQ.astro`, `pricing.astro`.

Exceptions:
- Touch targets for header nav links: minimum 44px height (achieved via `h-16` header = 64px, buttons with `py-2 px-4` satisfy 36px minimum — acceptable at desktop; no mobile-specific exception needed for Phase 12 scope)
- Pricing card grid gap: `gap-6` = 24px between cards on `md:grid-cols-3`

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 500 (medium) | 1.4 |
| Heading (section) | 30px (`text-3xl`) | 700 (bold) | 1.2 |
| Display (hero/pricing h1) | 36–48px (`text-4xl` / `text-5xl`) | 800 (extrabold) | 1.1 |

**Source:** Detected from existing components. `VCardTab.tsx` uses `text-sm font-medium` for labels. `Hero.astro` uses `text-4xl sm:text-5xl lg:text-6xl font-extrabold`. `FAQ.astro` uses `text-3xl font-bold` for section heading and `text-base font-semibold` for question text. `pricing.astro` uses `text-xl font-bold` for card titles.

**Phase 12 specific — vCard new fields:** All 6 new fields (Title, Company, Work Phone, Address, Website, LinkedIn) follow the same label/input pattern as existing VCardTab fields: `text-sm font-medium text-gray-700 dark:text-slate-300` labels, `text-sm` inputs with `px-3 py-2 border border-gray-300 rounded-md`.

**Source:** Detected from `global.css` (`--color-brand: #2563EB`), `Hero.astro` (`bg-gray-50`), `Layout.astro` (`dark:bg-[#0f172a]`), `pricing.astro` (`bg-[#2563EB]` for active toggle + Pro CTA button).

**Accent reserved for (explicit list):**
1. Primary CTA buttons: "Sign In", "Get Pro", "Get Starter", "Start for free" — filled (`bg-[#2563EB] text-white`) or outlined (`border-[#2563EB] text-[#2563EB]`)
2. Active toggle state on billing cadence selector (monthly/annual)
3. Focus rings on all form inputs (`focus:ring-blue-500 focus:border-blue-500`)

**Success state:** `text-green-500` / `text-green-600` for pricing feature checkmarks and annual savings badges. Carry forward unchanged.

## Component Inventory

These are the specific UI elements changed or added in Phase 12. All new elements extend existing patterns without introducing new visual primitives.

### Header (Header.astro)

**Current state:** Single "Sign In" button (filled blue) shown when signed out. No Pricing link.

**Required changes:**
- Add "Pricing" text link to nav. Position: between logo and auth buttons. Style: `text-sm font-medium text-gray-700 hover:text-gray-900 dark:text-slate-300 dark:hover:text-white`. No background, no border. This is a navigation link, not a CTA — accent color not applied.
- Add "Register" button shown alongside "Sign In" when signed out. Style: outlined variant — `border border-[#2563EB] text-[#2563EB] text-sm font-semibold px-4 py-2 rounded-lg hover:bg-blue-50 dark:border-blue-400 dark:text-blue-400 dark:hover:bg-slate-800 transition-colors focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2`. Href: `/signup`.
- "Sign In" button retains filled style. Order: Pricing link → Register button → Sign In button (left to right).

### Hero copy (Hero.astro)

**Current inaccurate copy:**
- Line: `"No signup, no limits — download as PNG or SVG instantly."` — contains "no limits" claim

**Required replacement:**
- `"No signup required — download as PNG or SVG instantly."` — removes inaccurate claim while preserving the conversion message. Approved by REQUIREMENTS.md COPY-01.

### FAQ copy (src/data/faq.ts or equivalent)

Any FAQ answer containing "no limits", "unlimited QR codes" without qualification, or freemium-inaccurate claims must be updated to reflect: Free plan = 5 saved QR codes, 3 dynamic. The exact FAQ item text is a content edit, not a structural change. Inline with existing `text-sm leading-relaxed text-gray-600 dark:text-slate-400` style.

### Pricing page (pricing.astro)

**Required structural changes (TIER-01, TIER-02, TIER-04):**

1. **Pro card:** Replace "Unlimited QR codes" list item with "250 QR codes". Replace checkmark icon color with `text-green-500` (consistent with other tiers).
2. **Starter card:** Add "10 dynamic QR codes" list item. "3 dynamic QR codes" already absent — no removal needed.
3. **Free card:** Add "3 dynamic QR codes" list item (shown as a limitation or simply omitted depending on product decision — default: omit dynamic QR row from Free card since Free users cannot use dynamic QRs).
4. **Starter card:** Remove "No ads" list item (`<li>` removal — TIER-04).
5. **Pro card:** Remove "No ads" list item if present (Pro card does not currently show "No ads" but confirm during implementation).

**No visual design changes** to the pricing grid layout, card borders, or toggle component. Pure content surgery.

### vCard tab (VCardTab.tsx)

**Required additions (VCARD-01):** Six new form fields added below the existing four (Full Name, Phone, Email, Organization). New fields in order:

| Field | Input type | Placeholder | Required |
|-------|-----------|-------------|----------|
| Title | text | "Software Engineer" | No |
| Company | text | "Acme Corp" | No |
| Work Phone | tel | "+1 555 000 0001" | No |
| Address | text | "123 Main St, City, State 12345" | No |
| Website | url | "https://example.com" | No |
| LinkedIn | url | "https://linkedin.com/in/username" | No |

All six use the identical CSS pattern as existing fields:
- Label: `block text-sm font-medium text-gray-700 dark:text-slate-300`
- Input: `w-full px-3 py-2 border border-gray-300 rounded-md shadow-sm text-sm focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 dark:bg-slate-800 dark:text-slate-100 dark:border-slate-600 dark:placeholder-slate-400`
- Wrapper: `space-y-2` for label+input pair; `space-y-4` stacks all pairs

**Note on "Company" vs "Organization":** The existing `org` field is labeled "Organization". The new Company field is a separate concept (company within an org, or an alias). Use "Company" as the label for the new `company` field in VCardState to avoid label collision. If the intent is to replace "Organization" with "Company", that is a product decision — default: add "Company" as a new field, leave "Organization" intact.
