## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Label-to-content gap (`space-y-2`), list item internal padding |
| md | 16px | Card internal horizontal padding (`px-4`), section horizontal padding |
| lg | 24px | Card internal padding (`p-6` = 24px), grid gap between cards |
| xl | 32px | Section vertical rhythm (`py-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level section spacing (`py-16`) |

**Source:** Inherited from Phase 12 UI-SPEC, detected from existing Tailwind usage in `Features.astro`, `FAQ.astro`, `Hero.astro`, and `pricing.astro`.

**Phase 13 exceptions:**
- Use case article grid gap: `gap-6` (24px) between cards, matching `Features.astro` grid pattern.
- How-to step number circle: 44px diameter (`w-11 h-11`) â€” minimum touch target for accessibility compliance.
- Screenshot images in how-to section: fixed height `h-48` (192px) on mobile, `h-64` (256px) on md+ screens, with `object-cover` to prevent layout shift.
- Pricing promo section: `gap-8` (32px) between feature list items to provide breathing room at the new conversion-focused placement.

## Accessibility Requirements

These are mandatory for all Phase 13 additions:

- **Breadcrumb:** `<nav aria-label="Breadcrumb">` wrapping an `<ol>`. Separator characters use `aria-hidden="true"`.
- **HowTo step list:** `<ol>` element with `role="list"` (removes list semantics in Safari VoiceOver per established pattern from `Features.astro`).
- **Use cases teaser grid:** `<ul role="list">`.
- **Screenshot `alt` text:** Descriptive, not generic. Example: `alt="Step 1: QR code generator showing the URL input tab selected"`, not `alt="screenshot"`.
- **CTA buttons:** All have visible focus rings (`focus:ring-2 focus:ring-blue-500`) and contrast ratio â‰Ą 4.5:1 (white text on `#2563EB` = 4.53:1 â€” passes AA).
- **Sitemap link rel:** `<link rel="sitemap" href="/sitemap-index.xml" />` â€” machine-readable only, no user-visible element needed.
