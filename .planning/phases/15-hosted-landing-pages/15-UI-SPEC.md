## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact button padding, label-to-input gap |
| md | 16px | Default field spacing, section internal padding |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout column gaps, major card breaks |
| 2xl | 48px | Landing page section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- File upload drop-zone minimum height: 96px (24 * 4) — large enough for drag target affordance
- Store button (App Store / Google Play) minimum touch target: 44px height — accessibility requirement for tappable links on mobile

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (normal) | 1.5 |
| Label | 14px | 500 (medium) | 1.4 |
| Heading | 20px | 700 (bold) | 1.2 |
| Display | 28px | 700 (bold) | 1.2 |

Source: matched from existing VCardTab.tsx (`text-sm font-medium`), QRLibrary.tsx (`text-xs font-semibold`), and FrameSection.tsx (`text-sm font-bold`). Four sizes declared; two weights (400 normal, 700 bold) with 500 medium used only on form labels.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff / slate-950 (dark) | Page background, form surfaces |
| Secondary (30%) | #f9fafb (gray-50) / slate-800 (dark) | Cards, upload drop-zones, file preview panels, sidebar |
| Accent (10%) | #2563EB (`--color-brand`) | Primary CTA buttons only; active tab indicator; focus rings (ring-blue-500) |
| Destructive | #dc2626 (red-600) | Delete landing page confirm button only |

Accent reserved for: "Create PDF Landing Page" primary button, "Create App Store Page" primary button, "Save QR Code" button, active form tab underline, input focus ring. Not applied to store buttons (platform-native colors used instead — see Store Buttons section).

Source: `src/styles/global.css` (`--color-brand: #2563EB`), VCardTab.tsx (`focus:ring-blue-500`), QRLibrary.tsx (`bg-red-600` for delete confirm).

## Platform Store Buttons

Store buttons on the `/p/[slug]` hosted landing page use platform-native colors, not the brand accent:

| Button | Background | Text | Logo |
|--------|-----------|------|------|
| App Store (iOS) | #000000 | #ffffff | Apple logo (inline SVG) |
| Google Play | #000000 | #ffffff | Google Play logo (inline SVG) |

Minimum height: 44px. Width: 160px minimum. Both buttons always rendered side by side on the landing page regardless of the visitor's device OS (per decision D-05).

## Component Inventory

### Generator Tabs (new — added to QRGeneratorIsland)

**PdfTab** — follows VCardTab.tsx pattern exactly:
- Container: `<div className="space-y-4">`
- Each field: `<div className="space-y-2">` wrapping label + input
- Label class: `block text-sm font-medium text-gray-700 dark:text-slate-300`
- Input class: `w-full px-3 py-2 border border-gray-300 rounded-md shadow-sm text-sm focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 dark:bg-slate-800 dark:text-slate-100 dark:border-slate-600 dark:placeholder-slate-400`
- Required fields marked: `<span className="text-red-500">*</span>` after label text
- Fields: Title*, Description* (textarea, 3 rows), Website URL, Company Name, CTA Button Text, Cover Photo upload, PDF File upload, Social Buttons (Facebook / Twitter / LinkedIn checkboxes)

**AppStoreTab** — same pattern as PdfTab:
- Fields: App Name*, Description* (textarea, 3 rows), Company Name, CTA Button Text, iOS App Store URL, Google Play URL, App Icon upload, Screenshot/Trailer URL

### File Upload Drop-Zone

Used for: Cover Photo (PDF page), PDF File (PDF page), App Icon (App Store page).

Visual contract:
- Default state: dashed border `border-2 border-dashed border-gray-300 dark:border-slate-600`, `rounded-lg`, min-height 96px, centered label text `text-sm text-gray-500 dark:text-slate-400`
- Drag-over state: `border-blue-400 bg-blue-50 dark:bg-blue-950/20` 
- File selected state: show filename + file size in `text-sm text-gray-700 dark:text-slate-300`, show a "Remove" link `text-xs text-red-500 hover:text-red-700`
- Upload in progress: show spinner (Lucide `Loader2` animated) + "Uploading…" text, disable further interaction
- Error state: `border-red-400 bg-red-50 dark:bg-red-950/20`, show error message below in `text-xs text-red-600 dark:text-red-400`
- File size limit label: shown as helper text below drop-zone — "Max 10 MB (Free) / 25 MB (Pro)" — using `text-xs text-gray-400 dark:text-slate-500`

### Social Sharing Toggle Group (PDF tab)

Three checkboxes rendered as pill-shaped toggle buttons: Facebook, Twitter/X, LinkedIn.

- Default (unchecked): `px-3 py-1 text-sm rounded-full border border-gray-300 dark:border-slate-600 text-gray-600 dark:text-slate-400`
- Checked: `px-3 py-1 text-sm rounded-full border border-blue-500 bg-blue-50 dark:bg-blue-950/40 text-blue-700 dark:text-blue-300 font-medium`
- Rendered as `<fieldset>` with `<legend className="sr-only">` for accessibility

### Hosted Landing Page (`/p/[slug]`)

The public-facing hosted page has two layouts.

**PDF Landing Page layout:**
- Full-width hero: cover photo at top, max-height 240px, `object-cover`, fallback background `bg-gray-100 dark:bg-slate-800` with centered file icon
- Content area: two-column on md+ screens — left: embedded PDF viewer (iframe or PDF.js), right: metadata sidebar
- Metadata sidebar: title (Display 28px/bold), company (Body 14px/400), description (Body 14px/400, line-height 1.5), website link (Body 14px/accent color), social sharing row
- Download button: secondary style — `border border-gray-300 dark:border-slate-600 px-4 py-2 rounded-md text-sm font-medium`

**App Store Landing Page layout:**
- App icon: 80px × 80px, `rounded-2xl`, centered at top with app name below (Display 28px/bold)
- Description: Body 14px/400, centered, max-width 480px
- Store buttons row: two side-by-side platform buttons (see Store Buttons above), centered, `gap-4`
- Screenshot/trailer URL: rendered as `<img>` preview or `<video>` beneath store buttons if provided

Both pages:
- Max-width container: `max-w-2xl mx-auto px-4 py-12`
- Background: white / dark:slate-950 (dominant surface)
- OG meta tags rendered server-side (SSR page)
- No navigation header — standalone, branded page only

### QR Library Card Extension (Dashboard)

Two new card body variants following the existing `DynamicCardBody` pattern:

**PdfCardBody:** shows "PDF Landing Page" badge (indigo pill, same as DynamicBadge pattern), landing page title truncated to 1 line, "View Page" link + "Edit" icon button.

**AppStoreCardBody:** shows "App Store" badge (emerald pill — `bg-emerald-100 text-emerald-700 dark:bg-emerald-900/40 dark:text-emerald-400`), app name truncated to 1 line, "View Page" link + "Edit" icon button.

Edit landing page action opens edit route (full-page edit, not modal) — accessible from the card actions row.

## Interaction States

### File Upload

| State | Visual |
|-------|--------|
| Idle | Dashed border, muted label |
| Drag-over | Blue tinted border + bg |
| Uploading | Spinner + "Uploading…", input disabled |
| Complete | Filename + size shown, "Remove" link |
| Error | Red tinted border, error message below |

### Form Submission

| State | Visual |
|-------|--------|
| Idle | Blue primary button active |
| Submitting | Button shows spinner + "Saving…", button disabled |
| Success | Toast "Landing page created" (sonner, same pattern as existing toasts) |
| Error | Toast "Could not save your landing page. Please try again." |

### Delete Confirmation (Dashboard)

Inline confirmation — no modal. When "Delete" is clicked, the card actions row replaces with "Are you sure?" text + "Yes, delete" (red) + "Cancel" buttons. Matches existing QRLibrary.tsx `deletingId` pattern exactly.
