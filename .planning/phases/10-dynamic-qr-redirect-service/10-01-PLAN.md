```typescript
import { sqliteTable, text, integer, index } from 'drizzle-orm/sqlite-core';

export const dynamicQrCodes = sqliteTable('dynamic_qr_codes', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  savedQrCodeId: text('saved_qr_code_id').notNull().references(() => savedQrCodes.id, { onDelete: 'cascade' }),
  userId: text('user_id').notNull(),
  slug: text('slug').notNull().unique(),
  destinationUrl: text('destination_url').notNull(),
  isPaused: integer('is_paused', { mode: 'boolean' }).notNull().default(false),
  createdAt: integer('created_at').notNull().$defaultFn(() => Math.floor(Date.now() / 1000)),
  updatedAt: integer('updated_at').notNull().$defaultFn(() => Math.floor(Date.now() / 1000)),
}, (table) => [
  index('dynamic_qr_codes_user_id_idx').on(table.userId),
]);
```

Key decisions:
- Separate table (not extending savedQrCodes) per research recommendation — cleaner for Phase 11 analytics FK
- `userId` denormalized for count queries without join (DYN-05 tier limit check)
- `savedQrCodeId` FK with cascade delete — when savedQrCode is deleted, dynamic record goes too
- `slug` has UNIQUE constraint (collision handled by retry in API layer)
- `isPaused` uses `integer mode: boolean` (same pattern as `cancelAtPeriodEnd` in subscriptions)

Create `src/pages/r/[slug].ts` — the redirect endpoint that scanned QR codes hit.

Implementation per D-15, D-16, D-17 and Research Pattern 2:

```typescript
export const prerender = false;

import type { APIRoute } from 'astro';
import { db } from '../../db/index';
import { dynamicQrCodes } from '../../db/schema';
import { eq } from 'drizzle-orm';
```

`GET` handler:
1. Extract `slug` from `params`. If missing, return `holdingResponse('invalid')`.
2. Query `dynamicQrCodes` selecting only `destinationUrl` and `isPaused` where `slug` matches. Use `.limit(1)`.
3. If no row found: return `holdingResponse('invalid')` (status 404).
4. If `row.isPaused` is true: return `holdingResponse('paused')` (status 200).
5. Otherwise: return `new Response(null, { status: 307, headers: { Location: row.destinationUrl } })`.

Use 307 (Temporary Redirect) — NOT 301. 301 is cached by browsers and would prevent destination updates from working.

`holdingResponse` helper function returns inline HTML. Use the exact code example from 10-RESEARCH.md (the `holdingResponse` function). The holding page must be:
- Mobile-first (people scan QR codes on phones)
- Dark mode via `prefers-color-scheme: dark` media query
- QRCraft text logo (not an image — avoids an extra request)
- Paused heading: "This QR code is temporarily paused." / body: "The owner has disabled this link."
- Invalid heading: "This QR code is no longer active." / body: "The link you scanned is no longer available."
- Status 200 for paused, 404 for invalid (per Research code example)
- `<html lang="en">` for accessibility, `<main>` landmark, heading as `<h1>`
- No nav, no footer, no external dependencies — completely self-contained HTML

Do NOT add `export const runtime = 'edge'` — it is not supported per-route in Astro 5 + @astrojs/vercel 9.x. The endpoint runs as a Vercel Serverless Function. Turso HTTP API provides the low latency.
