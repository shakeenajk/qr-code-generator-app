## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (gap-1, px-1) |
| sm | 8px | Compact element spacing (gap-2, p-2) |
| md | 16px | Default element spacing (p-4, gap-4) |
| lg | 24px | Section padding (mb-6, py-6) |
| xl | 32px | Layout gaps (py-8) |
| 2xl | 48px | Major section breaks (py-12) |
| 3xl | 64px | Page-level spacing (py-20 on empty states) |

Exceptions:
- Touch targets for toggle/action buttons: minimum 44px height (p-2.5 + text-sm achieves ~38px; use min-h-[44px] on mobile for pause/activate toggle button)
- Holding page (mobile-first): use pt-16 top offset to clear phone status bar, pb-8 bottom safe area

Source: pre-populated from existing QRLibrary.tsx pattern analysis (p-4 cards, gap-4 grid, mb-6 headers, py-20 empty state)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 500 (medium) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-2xl) | 700 (bold) | 1.15 |

Notes:
- Body/label distinction: body for secondary metadata (date, truncated URL), label for form field labels and button text
- Holding page uses Display (28px / 700) for the message headline; Body for sub-copy
- Inline edit affordance for destination URL: text-sm (14px) / weight 400 in read state, same in edit state
- Badge text ("Dynamic"): text-xs (12px) / weight 600

Source: pre-populated from existing patterns in QRLibrary.tsx (text-xl font-semibold for headings, text-sm for body, text-xs for metadata)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-white` / `dark:bg-[#0f172a]` | Page background, card backgrounds (`dark:bg-gray-800`) |
| Secondary (30%) | `bg-gray-50` / `dark:bg-slate-800` | Card interiors, input backgrounds, hover states |
| Accent (10%) | `#4F46E5` (indigo-600) | Reserved list below |
| Destructive | `#DC2626` (red-600) | Delete button, delete confirmation button only |

Accent (`indigo-600 / dark:indigo-500`) reserved for:
1. Active state of grid/list view toggle buttons (bg-indigo-600 text-white)
2. Primary CTA buttons ("Go to Generator", "Save" in inline edit)
3. Loading spinner border (border-indigo-600)
4. "Dynamic QR" toggle enabled state (the toggle thumb/track when on)
5. Focus ring on URL destination input (focus:ring-indigo-500)

Secondary semantic colors (status indicators — not accent):
- Active/enabled QR status dot: `bg-green-500` — use `w-2 h-2 rounded-full inline-block` before status label
- Paused QR status dot: `bg-amber-400` — same dot pattern
- Pro lock indicator on toggle: `text-purple-500` Lock icon (matches existing TierBadge purple for Pro)
- Upgrade prompt accent: `bg-indigo-600` (same as primary accent — upgrade is a desired action)

Source: pre-populated from QRLibrary.tsx (indigo-600 for active toggle, red-600 for delete), TierBadge.tsx (purple for Pro), global.css (`--color-brand: #2563EB` noted but indigo-600 is the established UI accent from existing components)

Note on brand color: `--color-brand` in global.css is `#2563EB` (blue-600). Existing components use `indigo-600` (#4F46E5) for interactive accent. Do NOT change existing components. Phase 10 new components use `indigo-600` to match established pattern.

## Component Inventory

### New Components (Phase 10)

#### 1. DynamicQRToggle (in UrlTab or QRGeneratorIsland)
- Location: Below URL input in UrlTab.tsx or QRGeneratorIsland.tsx URL section
- Layout: Row with toggle switch (left) + label "Dynamic QR" (center) + info hint icon (right)
- Toggle: HTML checkbox styled as pill toggle — `w-11 h-6 rounded-full` track, `w-5 h-5 rounded-full` thumb
- Enabled track color: `bg-indigo-600`; disabled track: `bg-gray-200 dark:bg-slate-700`
- Info hint: Lucide `Info` icon (`w-4 h-4 text-gray-400`) — hover/focus shows tooltip below
- Tooltip text: "Destination can be changed after printing without a new QR code."
- Pro lock state (limit reached): Lock icon (`w-3.5 h-3.5 text-purple-500`) replaces toggle; clicking shows upgrade toast
- Greyed-out state (non-URL tab): `opacity-40 cursor-not-allowed` on entire row; tooltip: "Dynamic QR only works with URL content."
- Dark mode: full `dark:` variant coverage

#### 2. DynamicBadge (on QR card)
- Pill badge: `text-xs font-semibold px-2 py-0.5 rounded-full bg-indigo-100 text-indigo-700 dark:bg-indigo-900/40 dark:text-indigo-400`
- Text: "Dynamic"
- Position: Inline after QR name in card header row (matches TierBadge positioning pattern)

#### 3. DynamicCardExtension (additions to existing card)
- Destination URL row: `text-xs text-gray-500 dark:text-slate-400 truncate max-w-[240px]` with edit icon
- Edit icon: Lucide `Pencil` (`w-3 h-3 text-gray-400 hover:text-gray-600 dark:hover:text-slate-300 cursor-pointer ml-1`)
- Status row: Status dot + status text in same `text-xs` size
- Active text: `text-green-600 dark:text-green-400`
- Paused text: `text-amber-600 dark:text-amber-400`

#### 4. InlineDestinationEditor (on QR card)
- Activated by: clicking Pencil icon next to destination URL
- Input: `w-full px-2 py-1 text-xs border border-indigo-500 rounded-md focus:outline-none focus:ring-1 focus:ring-indigo-500 dark:bg-slate-800 dark:text-slate-100`
- Action row below input: "Save" button (`text-xs px-2 py-1 rounded bg-indigo-600 text-white hover:bg-indigo-700`) + "Cancel" link (`text-xs text-gray-500 dark:text-slate-400 hover:text-gray-700 ml-2`)
- Loading state: "Save" button shows spinner, becomes disabled (`opacity-60 cursor-wait`)
- Max width: same as card body width (no horizontal overflow)

#### 5. PauseToggle (on QR card)
- Pill button: `text-xs px-2.5 py-1.5 rounded-md border transition-colors`
- Active state: `border-green-200 dark:border-green-800 text-green-700 dark:text-green-400 hover:bg-green-50 dark:hover:bg-green-950`
- Paused state: `border-amber-200 dark:border-amber-800 text-amber-700 dark:text-amber-400 hover:bg-amber-50 dark:hover:bg-amber-950`
- Icon: Lucide `Pause` (active) / `Play` (paused), `w-3.5 h-3.5`
- Text: "Pause" (when active) / "Activate" (when paused)
- No confirmation modal — immediate action with toast feedback
- Loading: button disabled + spinner while API in flight

#### 6. HoldingPage (`/r/[slug]` paused or invalid response)
- Layout: Full-page centered flex column, `min-h-screen bg-white dark:bg-[#0f172a]`
- No navigation, no footer — bare page
- QRCraft logo: centered, `w-32` image or text wordmark
- Message: Display typography (28px / 700) centered, `text-gray-900 dark:text-white`
- Sub-copy: Body typography (16px / 400), `text-gray-500 dark:text-slate-400`, centered, `max-w-xs`
- Padding: `px-6 py-16` (mobile-first)
- Dark mode supported

#### 7. UpgradePrompt (inline in generator for limit-hit users)
- Triggered: when free/Starter user tries to enable Dynamic QR toggle after 3 exist
- Delivery: Sonner toast (matches existing pattern) — not a modal
- Toast type: `toast.error()` with action button
- Toast content: "Upgrade to Pro for unlimited dynamic QR codes" + "Upgrade" action → navigates to `/pricing`

## Accessibility Contract

- DynamicQRToggle: `role="switch"` + `aria-checked={enabled}` + `aria-label="Enable Dynamic QR"` + `aria-disabled` when locked/greyed
- PauseToggle: `aria-label="Pause QR code"` / `aria-label="Activate QR code"` (switches with state)
- InlineDestinationEditor input: `aria-label="Destination URL"` + `aria-describedby` pointing to error text when present
- DynamicBadge: `aria-label="Dynamic QR code"` on the pill (or wrapped in visually-hidden span for screen readers)
- Status dots: `aria-hidden="true"` on dot element; text label carries meaning
- Holding page: `<main>` landmark with `lang` on `<html>`; heading is `<h1>`; no interactive elements needed
- All buttons: minimum 44px touch target via `min-h-[44px]` on mobile (Tailwind responsive: default `min-h-[44px]`, `sm:min-h-auto`)
