  <action>
    Create `src/pages/api/v1/generate.ts` with `export const prerender = false` as the FIRST line.

    **Request validation:**
    - Accept only POST method. Return 405 for other methods.
    - Parse JSON body. Expect shape:
      ```
      {
        content: string;                           // required, max 4096 chars
        contentType: 'url' | 'text' | 'wifi' | 'vcard';  // required
        format?: 'png' | 'svg';                   // default: 'png'
        size?: number;                             // default: 400, min: 100, max: 1000
        errorCorrectionLevel?: 'L' | 'M' | 'Q' | 'H';  // default: 'M'
      }
      ```
    - Validate `content` is a non-empty string, max 4096 chars. Return 400 with `{ error: "content is required and must be a string (max 4096 chars)" }` if invalid.
    - Validate `contentType` is one of the four allowed values. Return 400 if invalid.
    - Validate `format` is 'png' or 'svg' if provided. Default to 'png'.
    - Validate `size` is 100-1000 if provided. Default to 400.
    - Validate `errorCorrectionLevel` is L/M/Q/H if provided. Default to 'M'.

    **Authentication (call verifyApiKey from Plan 01):**
    ```typescript
    import { verifyApiKey } from '../../../lib/apiAuth';
    const verified = await verifyApiKey(request);
    if (!verified) {
      return new Response(JSON.stringify({ error: 'Invalid or revoked API key' }), {
        status: 401, headers: { 'Content-Type': 'application/json' }
      });
    }
    ```

    **Per-key rate limiting (call getApiKeyRateLimiter from Plan 01):**
    ```typescript
    import { getApiKeyRateLimiter } from '../../../lib/apiRateLimit';
    const limiter = getApiKeyRateLimiter();
    if (limiter) {
      const { success, reset } = await limiter.limit(verified.keyId);
      if (!success) {
        const retryAfter = Math.ceil((reset - Date.now()) / 1000);
        return new Response(JSON.stringify({ error: 'Rate limit exceeded', retryAfter }), {
          status: 429,
          headers: {
            'Content-Type': 'application/json',
            'Retry-After': String(retryAfter),
          },
        });
      }
    }
    ```

    **QR generation using `qrcode` npm (NOT qr-code-styling — DOM dependent):**
    ```typescript
    import QRCode from 'qrcode';
    ```
    - For PNG: `const dataUrl = await QRCode.toDataURL(body.content, { type: 'image/png', width: size, margin: 1, errorCorrectionLevel, color: { dark: '#000000', light: '#ffffff' } });` then strip prefix: `const base64 = dataUrl.split(',')[1];`
    - For SVG: `const svgString = await QRCode.toString(body.content, { type: 'svg', width: size, margin: 1, errorCorrectionLevel });`

    **Usage tracking (atomic increment per research Pitfall 5):**
    After successful generation, increment usageCount and update lastUsedAt:
    ```typescript
    import { db } from '../../../db/index';
    import { apiKeys } from '../../../db/schema';
    import { eq, sql } from 'drizzle-orm';

    await db.update(apiKeys)
      .set({
        usageCount: sql`${apiKeys.usageCount} + 1`,
        lastUsedAt: Math.floor(Date.now() / 1000),
      })
      .where(eq(apiKeys.id, verified.keyId));
    ```
    Use `sql` template for atomic increment — never read-then-write.

    **Success response:**
    ```typescript
    return new Response(JSON.stringify({
      format: format,
      data: format === 'png' ? base64 : svgString,
      contentType: format === 'png' ? 'image/png' : 'image/svg+xml',
    }), {
      status: 200,
      headers: { 'Content-Type': 'application/json' },
    });
    ```

    **Error handling:** Wrap QR generation in try/catch. On error, return 500 with `{ error: 'QR generation failed' }`. Do NOT expose internal error details.

    **Security reminders:**
    - Never log the raw API key from the Authorization header
    - Always return JSON content type on all responses (including errors)
    - The `export const prerender = false` is mandatory — static mode silently breaks this route
