       ```
       npm install qrcode && npm install -D @types/qrcode
       ```

    2. **Add apiKeys table to src/db/schema.ts** — append after the existing `scanEvents` table. Follow the exact Drizzle pattern from 19-RESEARCH.md Pattern "New Drizzle Schema":
       - `id`: text PK with `$defaultFn(() => crypto.randomUUID())`
       - `userId`: text not null
       - `name`: text not null (user-facing label)
       - `keyHash`: text not null unique (SHA-256 of raw key)
       - `keyPrefix`: text not null (first 12 chars of raw key — "qrc_" + 8 hex chars — safe for display)
       - `usageCount`: integer not null default 0
       - `lastUsedAt`: integer nullable (unix timestamp)
       - `revokedAt`: integer nullable (null = active; soft delete)
       - `createdAt`: integer not null with `$defaultFn(() => Math.floor(Date.now() / 1000))`
       - Indexes: `api_keys_user_id_idx` on userId, `api_keys_key_hash_idx` on keyHash (critical for every-request lookup)

    3. **Run Drizzle migration**:
       ```
       npx drizzle-kit generate
       npx drizzle-kit migrate
       ```

    4. **Update src/middleware.ts** — add Clerk exemption for /api/v1/* routes:
       - Add `const isPublicApiRoute = createRouteMatcher(['/api/v1/(.*)']);` after existing route matchers
       - Inside `clerkAuth`, add `if (isPublicApiRoute(context.request)) return;` after the webhook check and BEFORE the userId check
       - Do NOT change `shouldRateLimit()` — IP-based rate limiting still applies to /api/v1/* (both layers run; this is by design per research Pitfall 4)

    5. **Create src/lib/apiAuth.ts** — API key verification helper:
       - Export `interface VerifiedKey { userId: string; keyId: string; usageCount: number; }`
       - Export `async function verifyApiKey(request: Request): Promise<VerifiedKey | null>`
       - Extract Bearer token from Authorization header
       - Hash with `crypto.createHash('sha256').update(rawKey).digest('hex')`
       - Query `apiKeys` table: `WHERE keyHash = ? AND revokedAt IS NULL` (use `and(eq(apiKeys.keyHash, keyHash), isNull(apiKeys.revokedAt))`)
       - Return `{ userId, keyId: row.id, usageCount: row.usageCount }` or null
       - Import crypto from 'node:crypto'

    6. **Create src/lib/apiRateLimit.ts** — per-key rate limiter:
       - Export `function getApiKeyRateLimiter(): Ratelimit | null`
       - Module-level singleton pattern (mirror rateLimit.ts exactly)
       - `Ratelimit.slidingWindow(100, '60 s')` — 100 req/min per key
       - `prefix: 'qrcraft:api-key-rl'` (distinct from IP limiter's `'qrcraft:rl'`)
       - `analytics: true`
       - Null fallback when env vars missing (same pattern as rateLimit.ts)
  </action>

<task type="auto">
  <name>Task 2: Dashboard API key CRUD routes</name>
  <files>src/pages/api/dashboard/api-keys.ts, src/pages/api/dashboard/api-keys/[id].ts</files>

  <action>
    1. **Create src/pages/api/dashboard/api-keys.ts** with `export const prerender = false`:

       **GET handler** — list keys for the authenticated user:
       - Extract `userId` from `Astro.locals.auth()` — return 401 if not authenticated
       - Query `SELECT id, name, keyPrefix, usageCount, lastUsedAt, revokedAt, createdAt FROM apiKeys WHERE userId = ? ORDER BY createdAt DESC`
       - Return JSON array (never include keyHash in the response)

       **POST handler** — create a new API key:
       - Extract `userId` from `Astro.locals.auth()` — return 401 if not authenticated
       - Check user's tier: query subscriptions table, verify tier is 'pro'. If not pro, return 403 with `{ error: "API keys require a Pro subscription" }`
       - Check key count: `SELECT COUNT(*) FROM apiKeys WHERE userId = ? AND revokedAt IS NULL`. If >= 10, return 400 with `{ error: "Maximum 10 active API keys per account" }`
       - Parse `{ name }` from request body. Validate name is 1-50 chars. Return 400 if invalid.
       - Generate raw key: `const rawKey = 'qrc_' + crypto.randomBytes(32).toString('hex')`
       - Hash: `const keyHash = crypto.createHash('sha256').update(rawKey).digest('hex')`
       - Prefix: `const keyPrefix = rawKey.slice(0, 12)`
       - Insert into apiKeys table: `{ userId, name, keyHash, keyPrefix, usageCount: 0 }`
       - Return 201 with `{ key: rawKey, keyPrefix, name, id }` — rawKey shown once and NEVER AGAIN
       - Import crypto from 'node:crypto'

    2. **Create src/pages/api/dashboard/api-keys/[id].ts** with `export const prerender = false`:

       **DELETE handler** — revoke (soft delete) a key:
       - Extract `userId` from `Astro.locals.auth()` — return 401 if not authenticated
       - Get `id` from `Astro.params.id`
       - Update: `SET revokedAt = Math.floor(Date.now() / 1000) WHERE id = ? AND userId = ?` (IDOR protection: always scope by userId)
       - If no rows affected, return 404 `{ error: "Not found" }`
       - Return 200 `{ ok: true }`

    Both routes use the standard Astro APIRoute pattern. Always scope queries by userId to prevent IDOR (per research Pitfall 6).
  </action>
