Extend the existing BulkGenerateIsland.tsx with three capabilities: QR generation, thumbnail preview, and ZIP download.

**1. QR Generation Loop (wired to "Generate All" button):**

Import `QRCodeStyling` from `qr-code-styling` (dynamic import to avoid SSR issues — use `const QRCodeStyling = (await import('qr-code-styling')).default;` inside the generation function).

Create an async `handleGenerate()` function:
- Set `status` to `'generating'`, `isGenerating` to `true`
- Clear any previous `generatedBlobs` and revoke old preview object URLs
- Import QRCodeStyling dynamically
- Loop through `parsedRows` sequentially (NOT Promise.all):
  - For each row, determine QR data:
    - If columnType is 'url': use the `url` column value
    - If columnType is 'text': use the `text` column value
    - If columnType is 'wifi': encode as `WIFI:T:WPA;S:${ssid};P:${password};;`
  - Create `new QRCodeStyling({ width: 512, height: 512, type: "canvas", data: qrData, dotsOptions: { type: "rounded", color: "#1e293b" }, backgroundOptions: { color: "#ffffff" } })`
  - Call `await qr.getRawData("png")` to get the Blob
  - Determine filename: use `name` column if present, otherwise `qr-${i + 1}`; sanitize by replacing non-alphanumeric chars with `-`; append `.png`
  - Push `{ name, blob }` to results array
  - Create object URL from blob and append to previews state for incremental rendering
  - Update progress: `setProgress({ current: i + 1, total: parsedRows.length })`
  - Every CHUNK_SIZE (10) rows, yield: `await new Promise<void>(r => setTimeout(r, 0))`
- After loop: set `generatedBlobs` to results, set `status` to `'complete'`, set `isGenerating` to `false`

Wire the "Generate All" button (currently disabled from Plan 01) to call `handleGenerate()`. Button should show `<Loader2 className="animate-spin" />` and "Generating..." text while `isGenerating` is true.

**2. Progress Bar:**

When `status === 'generating'`, render above the preview grid:
- A horizontal progress bar: `<div className="h-2 bg-gray-200 rounded-full"><div className="h-full bg-blue-600 rounded-full transition-all" style={{ width: `${(progress.current / progress.total) * 100}%` }} /></div>`
- Text: "Generating {current} of {total}..."

**3. Thumbnail Preview Grid:**

When previews exist (during or after generation), render a responsive grid:
- `grid grid-cols-3 sm:grid-cols-4 md:grid-cols-6 lg:grid-cols-8 gap-3`
- Each item: `<div className="aspect-square bg-white rounded-lg border shadow-sm p-1"><img src={preview.src} alt={preview.name} className="w-full h-full object-contain" /></div>`
- Below each thumbnail: filename truncated to ~12 chars with `truncate` class
- Grid should be scrollable with `max-h-[400px] overflow-y-auto`

**4. ZIP Download:**

When `status === 'complete'`, show a prominent download button:
- Import JSZip from `jszip`
- Create `handleDownload()` async function:
  - `const zip = new JSZip();`
  - Loop through `generatedBlobs`: `zip.file(item.name, item.blob);`
  - `const zipBlob = await zip.generateAsync({ type: "blob" });`
  - Create anchor: `const url = URL.createObjectURL(zipBlob); const a = document.createElement('a'); a.href = url; a.download = 'qrcodes.zip'; a.click(); URL.revokeObjectURL(url);`
- Button text: "Download ZIP ({count} QR codes)" with a download icon from lucide-react

**5. Memory cleanup:**

- Add `useEffect` cleanup that revokes all preview object URLs on unmount
- When user uploads a new CSV (or clicks "Clear"), revoke all existing preview URLs before clearing state
- After ZIP download anchor click, immediately `URL.revokeObjectURL(url)` for the ZIP blob

**6. Cancel generation:**

Add a "Cancel" button visible during generation. Set a `cancelRef = useRef(false)`. In the generation loop, check `cancelRef.current` after each row — if true, break out of the loop, set status to 'parsed', clean up partial results.
