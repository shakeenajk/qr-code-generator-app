## Gaps Found

### 1. BRAND-04: Dark Mode Missing on Features and FAQ Sections

**Severity:** Gap — visual break in dark mode
**Requirement:** Site renders correctly in dark mode based on system preference
**Phase:** 04-export-and-launch

`Features.astro` and `FAQ.astro` have **zero `dark:` classes**. Both sections render with `bg-gray-50`, `bg-white` card backgrounds and `text-gray-900`/`text-gray-600` text — fixed light-mode colors. In dark mode, these sections render as bright white/gray boxes sitting between the dark Hero section and dark Footer, breaking visual continuity.

The `export.spec.ts` dark mode Playwright tests cover `body`, `header`, and `QRPreview` parent — Features and FAQ are untested.

Additionally, `REQUIREMENTS.md` checkbox for BRAND-04 is unchecked `[ ]` and the traceability table shows `Pending`.

**Fix required:**
- Add `dark:` classes to `Features.astro` (section bg, card bgs, text colors)
- Add `dark:` classes to `FAQ.astro` (section bg, item bgs, text colors)
- Add Playwright dark mode smoke tests for Features and FAQ sections
- Update REQUIREMENTS.md BRAND-04 checkbox to `[x]` and traceability to `Complete`

### 2. PREV-03: WiFi and vCard Tabs Show QR Instead of Ghost Placeholder on Empty State

**Severity:** Gap — behavioral inconsistency
**Requirement:** Preview shows placeholder when no content is entered
**Phase:** 02-core-generator

`isContentEmpty()` in `qrEncoding.ts` checks `!data.trim()` on the fully-encoded string. But `encodeWifi({ssid:"", password:"", security:"WPA"})` returns `"WIFI:T:WPA;S:;P:;;"` — a non-empty string — and `encodeVCard({name:"",phone:"",email:"",org:""})` returns a full `BEGIN:VCARD...END:VCARD` skeleton. Both are non-empty, so `isEmpty = false`, the ghost placeholder stays hidden, and a QR renders from a protocol-only string.

URL and Text tabs behave correctly — clearing the input produces `isEmpty = true` and shows the ghost.

The Phase 02 smoke test for PREV-03 only exercises the URL tab.

**Fix required:**
- Update `isContentEmpty()` to detect empty WiFi/vCard state before encoding (check raw field values, not the encoded string)
- Or add per-type emptiness logic in `QRGeneratorIsland.tsx` before computing `rawContent`
- Add PREV-03 smoke test coverage for WiFi and vCard empty states
