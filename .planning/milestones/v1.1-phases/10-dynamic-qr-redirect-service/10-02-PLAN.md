**Part A: Extend [id].ts** (per D-08, D-09, D-10)

Add a `PATCH` export to `src/pages/api/qr/[id].ts` for dynamic QR-specific updates. Keep existing GET/PUT/DELETE unchanged.

PATCH handler:
1. Auth check (same pattern as existing routes).
2. Parse JSON body. Accept `destinationUrl` (string) and/or `isPaused` (boolean).
3. Look up the dynamicQrCode row by `savedQrCodeId = id AND userId = userId` (IDOR prevention — same compound WHERE pattern as PUT/DELETE).
4. If no row found: return 404 `{ error: 'Not found' }`.
5. Build update object:
   - If `destinationUrl` provided: validate it is a non-empty string. Update `destinationUrl`.
   - If `isPaused` provided: update `isPaused`.
   - Always update `updatedAt` to current epoch seconds.
6. Execute update: `db.update(dynamicQrCodes).set(updates).where(and(eq(dynamicQrCodes.savedQrCodeId, id), eq(dynamicQrCodes.userId, userId)))`.
7. Return 200 `{ ok: true }`.

Also update the DELETE handler: Before deleting from `savedQrCodes`, explicitly delete from `dynamicQrCodes` where `savedQrCodeId = id AND userId = userId`. This is a safety belt in case FK cascade is not enforced by Turso's HTTP mode (Research Open Question #3). The explicit delete is harmless if cascade works, and essential if it doesn't.

```typescript
// In DELETE handler, before the existing savedQrCodes delete:
await db.delete(dynamicQrCodes).where(
  and(eq(dynamicQrCodes.savedQrCodeId, id), eq(dynamicQrCodes.userId, userId))
);
```

**Part B: Extend list.ts** (per D-06, D-07)

Modify `GET /api/qr/list` to include dynamic QR metadata via a LEFT JOIN.

1. Import `dynamicQrCodes` from schema and `leftJoin` support.
2. Replace the current `db.select({...}).from(savedQrCodes)` with a left join:
   ```typescript
   const rows = await db
     .select({
       id: savedQrCodes.id,
       userId: savedQrCodes.userId,
       name: savedQrCodes.name,
       contentType: savedQrCodes.contentType,
       contentData: savedQrCodes.contentData,
       styleData: savedQrCodes.styleData,
       thumbnailData: savedQrCodes.thumbnailData,
       createdAt: savedQrCodes.createdAt,
       updatedAt: savedQrCodes.updatedAt,
       // Dynamic QR fields (null for static QRs)
       slug: dynamicQrCodes.slug,
       destinationUrl: dynamicQrCodes.destinationUrl,
       isPaused: dynamicQrCodes.isPaused,
     })
     .from(savedQrCodes)
     .leftJoin(dynamicQrCodes, eq(savedQrCodes.id, dynamicQrCodes.savedQrCodeId))
     .where(eq(savedQrCodes.userId, userId))
     .orderBy(desc(savedQrCodes.createdAt))
     .limit(50);
   ```
3. The response shape now includes optional `slug`, `destinationUrl`, `isPaused` fields — they are `null` for static QRs and populated for dynamic QRs.
4. Add a computed `isDynamic` field to each row in the JSON response: `isDynamic: row.slug !== null`.
