Modify QRGeneratorIsland to manage `isDynamic` state and wire it to UrlTab and SaveQRModal. Per D-01, D-03, D-05, D-14, and Research Pattern 5.

1. **New state variables in QRGeneratorIsland:**
   ```typescript
   const [isDynamic, setIsDynamic] = useState(false);
   const [dynamicCount, setDynamicCount] = useState<number | null>(null);
   const [savedSlug, setSavedSlug] = useState<string | null>(null);
   ```

2. **Fetch dynamic QR count** for the current user (needed for limit check on toggle). When user is signed in and tier is not 'pro', fetch GET /api/qr/list and count items where `isDynamic === true`. Cache this count in `dynamicCount`. Do this once on mount (or when auth state resolves). If the count is >= 3, set `dynamicLocked = true`.

   ```typescript
   const dynamicLocked = userTier !== 'pro' && userTier !== null && isSignedIn && (dynamicCount ?? 0) >= 3;
   ```

3. **Toggle handler:**
   ```typescript
   const handleToggleDynamic = (enabled: boolean) => {
     if (dynamicLocked) {
       toast.error('Upgrade to Pro for unlimited dynamic QR codes', {
         action: { label: 'Upgrade to Pro', onClick: () => window.location.href = '/pricing' },
       });
       return;
     }
     if (!isSignedIn) {
       toast.error('Sign in to create dynamic QR codes');
       return;
     }
     setIsDynamic(enabled);
   };
   ```

4. **Pass new props to UrlTab:**
   ```tsx
   <UrlTab
     value={urlValue}
     onChange={setUrlValue}
     isDynamic={isDynamic}
     onToggleDynamic={handleToggleDynamic}
     dynamicLocked={dynamicLocked}
     isUrlTab={activeTab === 'url'}
   />
   ```

   Important: The UrlTab is always rendered (tabs preserve state), but the toggle's `isUrlTab` prop controls the greyed-out state per D-02.

5. **QR data encoding:** When `isDynamic` is true and `activeTab === 'url'`:
   - The QR preview encodes `https://qr-code-generator-app.com/r/--------` (placeholder) instead of the raw URL.
   - After save (when slug is returned), update contentData to include the real slug for edit-mode hydration.
   - Modify the data computation (where `qrCode.update({ data: ... })` is called):
   ```typescript
   const qrData = isDynamic && activeTab === 'url'
     ? `https://qr-code-generator-app.com/r/${savedSlug || '--------'}`
     : /* existing data computation */;
   ```

6. **Pass isDynamic and destinationUrl as props to SaveQRModal.** When `isDynamic` is true:
   - Pass `isDynamic={isDynamic}` and `destinationUrl={urlValue}` as new props to SaveQRModal.
   - SaveQRModal uses these to append `isDynamic: true` and `destinationUrl` to the POST body when calling `/api/qr/save`.

7. **Update SaveQRModal.tsx** to accept and use the new props:
   - Add `isDynamic?: boolean` and `destinationUrl?: string` to SaveQRModal's props interface.
   - In the fetch body construction, when `isDynamic` is true, add `isDynamic: true` and `destinationUrl` to the body object.
   - On success response, if `isDynamic` is true, extract `slug` from the response JSON and pass it back to the parent via an `onSaveSuccess` callback (or return it). The parent (QRGeneratorIsland) calls `setSavedSlug(slug)`.
   - Change the modal's save button label to "Save Dynamic QR" when `isDynamic` is true (per UI-SPEC copywriting contract).

8. **Do not render the toggle for anonymous users.** The toggle should only appear when `isSignedIn` is true. Anonymous users see the plain UrlTab without the toggle row. This avoids confusing anonymous users with a feature they can't use. Pass a prop like `showDynamicToggle={isSignedIn}` or conditionally pass the toggle props.

9. **Reset isDynamic when switching away from URL tab:** Do NOT reset isDynamic — the toggle just appears greyed out (per D-02). The user must explicitly turn it off. But if the user saves from a non-URL tab, isDynamic should be ignored (contentType won't be 'url', server rejects).
