**Prerequisites (before starting):**
1. Confirm `.env` has all Stripe env vars: `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, all 4 `STRIPE_PRICE_*` vars, `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`, `PUBLIC_BASE_URL=http://localhost:4321`
2. In a separate terminal: `stripe listen --forward-to localhost:4321/api/webhooks/stripe` (outputs a whsec_xxx — this must match your STRIPE_WEBHOOK_SECRET in .env)
3. Start dev server: `npm run dev`

**Automated DB setup:** Run `npx drizzle-kit push` to apply schema to Turso (or use `npx drizzle-kit migrate` if drizzle.config.ts exists).

**Test sequence:**

1. **Pricing page** — Visit http://localhost:4321/pricing
   - Expected: 3 tier cards visible (Free, Starter, Pro)
   - Expected: Pro card has ring/border highlight + "Most Popular" badge
   - Expected: Monthly/annual toggle switches prices

2. **Hero mention** — Visit http://localhost:4321
   - Expected: Small "Free forever · Pro from $3.99/mo" line below hero subtitle with link to /pricing

3. **Dashboard upgrade CTA** — Sign in, visit http://localhost:4321/dashboard
   - Expected: "Upgrade to Pro" panel visible at bottom of sidebar (as a free user)
   - Expected: No tier badge in UserMenu (free users show nothing)
   - Expected: No payment failure banner

4. **Checkout flow** — Click "Upgrade to Pro" in sidebar
   - Expected: Redirected to Stripe Checkout page (stripe.com)
   - Use Stripe test card: `4242 4242 4242 4242`, any future expiry, any CVC
   - Complete checkout
   - Expected: Redirected to http://localhost:4321/dashboard?upgraded=true
   - Expected: "Activating your plan..." indicator appears
   - Expected: Within 5 seconds, success toast "Welcome to Pro! Your plan is now active."
   - Expected: ?upgraded=true removed from URL

5. **Post-upgrade dashboard** — After toast dismisses
   - Expected: Sidebar shows "Manage subscription" link (not upgrade CTA)
   - Expected: UserMenu dropdown shows purple "Pro" pill badge

6. **Stripe Customer Portal** — Click "Manage subscription"
   - Expected: Redirected to Stripe Customer Portal
   - (Do not cancel — just verify portal loads correctly)

7. **Webhook verification** — In stripe listen terminal
   - Expected: Logs show `checkout.session.completed` → 200 response
   - Expected: `customer.subscription.updated` → 200 response
