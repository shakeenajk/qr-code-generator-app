Replace the current middleware with the updated version that adds `isWebhookRoute` exclusion. The existing middleware only protects `/dashboard(.*)` — API routes are not protected. But Stripe sends unauthenticated POST requests and explicit exclusion documents the intent and prevents future regressions.

```typescript
// src/middleware.ts
import { clerkMiddleware, createRouteMatcher } from '@clerk/astro/server';

const isProtectedRoute = createRouteMatcher(['/dashboard(.*)']);
const isWebhookRoute = createRouteMatcher(['/api/webhooks/(.*)']);

export const onRequest = clerkMiddleware((auth, context) => {
  // Webhooks must bypass auth — Stripe sends unauthenticated POST requests
  if (isWebhookRoute(context.request)) return;

  const { userId } = auth();
  if (!userId && isProtectedRoute(context.request)) {
    return context.redirect('/login');
  }
});
```

This is the exact pattern from RESEARCH.md (Pattern: Middleware: Exclude Webhook from Auth).

    - Duplicate event ID → silently return (no DB write), still 200
    - checkout.session.completed with subscription mode → upsert subscriptions row (userId from session.metadata.clerkUserId)
    - customer.subscription.updated → re-sync tier/status/priceId/cancelAtPeriodEnd from subscription object
    - customer.subscription.deleted → set tier='free', status='canceled'
    - invoice.paid → set status='active' (confirms payment)
    - invoice.payment_failed → set status='past_due'
    - customer.subscription.trial_will_end → log only (no DB change needed at trial_will_end stage)
    - Unknown event types → silently ignored (no error thrown)
  </behavior>
  <action>
Create `src/pages/api/webhooks/stripe.ts`. The file must have `export const prerender = false` at the top.

Follow RESEARCH.md Patterns 1 and 2 exactly.

Key implementation rules:
1. Use `await request.text()` for raw body — do this BEFORE any other body parsing. This is required for `stripe.webhooks.constructEvent()`.
2. Deduplication: `db.insert(stripeEvents).values({ eventId: event.id })` — if the INSERT throws (unique constraint = already processed), catch silently and return.
3. Return 200 AFTER processing (not before) — this is an Astro SSR route, not Express. Astro's runtime doesn't have the timeout concern Express does. Process synchronously.
4. `handleCheckoutCompleted`: retrieve full subscription via `stripe.subscriptions.retrieve(session.subscription)` to get priceId. Use `onConflictDoUpdate` targeting `subscriptions.userId`.
5. `handleSubscriptionUpdated` and `handleSubscriptionDeleted`: use `onConflictDoUpdate` or `update(...).where(eq(subscriptions.stripeSubscriptionId, sub.id))`.
6. For `invoice.paid` and `invoice.payment_failed`: update status only (do not change tier) — find row by `stripeCustomerId` from `invoice.customer`.
7. `clerkUserId` must come from `session.metadata.clerkUserId` (set at checkout session creation in Plan 03). If missing, log error and return without DB write.
