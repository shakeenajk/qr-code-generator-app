## Component Inventory

### AdUnit Component (`src/components/AdUnit.tsx`)

The only new UI surface in Phase 16. A tier-gated React component that renders exclusively for `isSignedIn && userTier === "free"`.

**Render conditions:**
- `userTier === null` (loading): render nothing — zero whitespace, no placeholder
- `userTier === "starter"` or `"pro"`: render nothing — zero whitespace, no placeholder
- `!isSignedIn` (anonymous): render nothing — zero whitespace, no placeholder
- `userTier === "free"` and `isSignedIn`: render the ad container (described below)

**Ad container visual contract:**
- Outer wrapper: `w-full mt-4` with `style={{ minHeight: 90 }}` — Ad container `minHeight: 90` matches Google AdSense standard leaderboard height; not a spacing token. Always reserves this height once rendered to prevent CLS.
- Disclosure label above ad: `text-xs text-gray-400 dark:text-slate-500 mb-1` — text: "Advertisement". `text-xs` (12px) is an existing codebase pattern for helper text (e.g. ExportButtons.tsx); not a typography contract entry.
- `<ins className="adsbygoogle" style={{ display: "block" }}>` — Google AdSense slot
- Ad format: `data-ad-format="horizontal"` with `data-full-width-responsive="true"` (responsive leaderboard)
- No border, no background, no shadow on the wrapper — the ad itself provides its own visual treatment

**Placement in `QRGeneratorIsland.tsx`:**
- Inside the preview panel (`lg:w-2/5` sticky column), after `<ExportButtons>` and the Save QR / Save Dynamic buttons
- Before the closing `</div>` of the preview column
- Conditional rendering guard: `{isSignedIn && userTier === "free" && <AdUnit adClient={...} adSlot={...} />}`

**CLS prevention — unfilled slot:**
Add to `src/styles/global.css`:
```css
ins.adsbygoogle[data-ad-status="unfilled"] {
  display: none !important;
}
```
The outer wrapper keeps `minHeight: 90` as stable reserved space. The "Advertisement" disclosure label remains visible above the collapsed `<ins>`.

**Script loading — delayed injection (no render-blocking):**
- `adsbygoogle.js` script is NOT added to `<head>` or any layout file
- Script is injected dynamically inside `AdUnit`'s `useEffect`, triggered by first user interaction (keydown, mousemove, wheel, touchmove, touchstart, scroll) OR a 5-second timeout fallback
- Script injection is guarded by `useRef(false)` to prevent double-injection under React Strict Mode

**Environment variables:**
- `PUBLIC_ADSENSE_CLIENT` — AdSense publisher ID (e.g. `ca-pub-XXXXXXXXXXXXXXXX`)
- `PUBLIC_ADSENSE_SLOT` — Ad unit slot ID (e.g. `1234567890`)
- Both accessed via `import.meta.env.PUBLIC_ADSENSE_CLIENT` / `import.meta.env.PUBLIC_ADSENSE_SLOT` in the React component

## Copywriting Contract

| Element | Copy |
|---------|------|
| Ad disclosure label | "Advertisement" |
| No other UI copy | Phase 16 adds no new headings, empty states, or user-facing error messages |

**Note on error states:** If the AdSense script fails to load (network error, ad blocker), the `<ins>` element renders empty with `data-ad-status="unfilled"` — the CSS rule hides it silently. No error message is shown to the user. This is intentional: ad loading failures are non-critical and should not interrupt the QR generation flow.

**No destructive actions** in Phase 16. No confirmation dialogs.

Source: RESEARCH.md AdSense TOS labeling requirement ("Advertisements" is an accepted disclosure label per Google AdSense Program policies). All other UI copy remains unchanged from prior phases.

## Interaction States

### AdUnit Component

| State | Condition | Visual |
|-------|-----------|--------|
| Hidden | `userTier !== "free"` or `!isSignedIn` or `userTier === null` | Nothing rendered — zero height, no whitespace |
| Pending (tier resolved, ad not yet loaded) | `userTier === "free"`, interaction not yet triggered | 90px reserved space + "Advertisement" label above empty `<ins>` |
| Script loading | First interaction event fired; `adsbygoogle.js` fetching | 90px reserved space + label; `<ins>` still empty |
| Ad served | AdSense returns creative | Ad renders inside `<ins>` within the 90px space |
| Unfilled | AdSense returns no ad (`data-ad-status="unfilled"`) | `<ins>` hidden via CSS; 90px wrapper + label still visible |

**No loading spinner** for the ad unit. Blank reserved space is the correct treatment — a spinner would draw attention to a non-critical load event.
