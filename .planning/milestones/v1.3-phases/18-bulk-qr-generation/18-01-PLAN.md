This is the main React island. For Plan 01, implement CSV upload, parsing, and tier enforcement. Leave generation/preview/ZIP for Plan 02.

Props interface:
```typescript
interface BulkGenerateIslandProps {
  tier: 'free' | 'starter' | 'pro';
}
```

State:
- `csvFile: File | null` — the uploaded CSV file
- `parsedRows: Record<string, string>[]` — parsed CSV rows
- `parseError: string | null` — CSV parsing error message
- `columnType: 'url' | 'text' | 'wifi' | null` — detected column type
- `generatedBlobs: { name: string; blob: Blob }[]` — filled by Plan 02
- `isGenerating: boolean` — generation in progress flag
- `progress: { current: number; total: number }` — generation progress
- `status: 'idle' | 'parsed' | 'generating' | 'complete'` — workflow state

Implementation:
1. **Free tier gate:** If `tier === 'free'`, render an upgrade CTA panel (styled box with "Upgrade to Starter or Pro to unlock bulk QR generation" message and a link to `/pricing`). Do NOT show upload controls.

2. **CSV upload area:** Use a drop zone similar to FileUploadZone pattern (drag-and-drop + click to select). Accept `.csv` files only (`accept=".csv"`). Max file size 5 MB.

3. **CSV parsing with papaparse:** On file select, call:
   ```typescript
   Papa.parse<Record<string, string>>(file, {
     header: true,
     skipEmptyLines: true,
     complete: (results) => { /* validate and set state */ },
     error: (err) => { /* set parseError */ },
   });
   ```

4. **Column normalization:** After parsing, normalize all column keys to lowercase and trim whitespace. Detect column type:
   - If `url` column exists → type is 'url'
   - If `text` column exists → type is 'text'
   - If `ssid` column exists → type is 'wifi'
   - If none match → set parseError "CSV must contain a 'url', 'text', or 'ssid' column"

5. **Tier limit check:** After parsing, compare `parsedRows.length` against `BULK_TIER_LIMITS[tier]`. If exceeded, show toast.error via sonner: `"Your ${tier} plan supports up to ${cap} rows. Upload a smaller CSV or upgrade."` and do NOT set parsedRows to state.

6. **Parsed rows summary table:** When status is 'parsed', show:
   - Row count: "X rows detected (type: URL/Text/WiFi)"
   - A scrollable table showing first 5 rows as preview with column headers
   - A "Generate All" button (disabled — will be wired in Plan 02)
   - A "Clear" button to reset state

7. **Provide a downloadable CSV template:** Below the upload zone, add three links:
   - "Download URL template" → triggers download of a CSV with headers `url,name` and 2 example rows
   - "Download Text template" → headers `text,name` with 2 example rows
   - "Download WiFi template" → headers `ssid,password,name` with 2 example rows
   Use `URL.createObjectURL(new Blob([csvContent], { type: 'text/csv' }))` with an anchor click.

Style with Tailwind following existing dashboard patterns. Use `sonner` toast for errors.
