  <files>src/lib/contrastUtils.ts</files>
  <behavior>
    - contrastRatio('#000000', '#ffffff') returns ~21 (maximum contrast)
    - contrastRatio('#ffffff', '#ffffff') returns 1 (no contrast)
    - contrastRatio('#1e293b', '#ffffff') returns > 4.5 (default Phase 2 colors pass)
    - isLowContrast('#1e293b', '#ffffff') returns false (defaults pass)
    - isLowContrast('#ffff00', '#ffffff') returns true (yellow on white fails 4.5:1)
    - isLowContrast('#ffffff', '#000000') returns false (same as black/white, args swapped)
    - contrastRatio is symmetric: contrastRatio(a, b) === contrastRatio(b, a)
  </behavior>
  <action>
Create `src/lib/contrastUtils.ts` implementing the WCAG 2.1 relative luminance formula exactly as specified in 03-RESEARCH.md Pattern 3. Export two functions:

1. `contrastRatio(hex1: string, hex2: string): number`
   - Parse hex colors with `hexToRgb` helper (handles `#rrggbb` 6-char format)
   - Compute relative luminance per WCAG formula: linearize each channel (≤0.03928: s/12.92; else: ((s+0.055)/1.055)^2.4), then L = 0.2126*R + 0.7152*G + 0.0722*B
   - Return (lighter + 0.05) / (darker + 0.05)

2. `isLowContrast(fgHex: string, bgHex: string): boolean`
   - Returns `contrastRatio(fgHex, bgHex) < 4.5`
   - 4.5:1 is WCAG AA threshold for normal text (per discretion recommendation in CONTEXT.md)

Write tests in `src/lib/contrastUtils.test.ts` using Vitest (check package.json for test runner — if Vitest not installed, write tests inline as assertions using `console.assert` since the project uses Playwright not Vitest). Verify behavior list above.

Note: No new npm packages — pure math, browser/Node compatible.

export type ColorSectionState = {
  dotColor: string;          // hex, default "#1e293b"

**Component structure:**
```
<section aria-labelledby="colors-heading">
  <h3 id="colors-heading" className="text-sm font-semibold text-gray-900 mb-3">Colors</h3>

  <!-- Foreground color picker (CUST-01) -->
  <div data-testid="color-fg">
    <ColorPicker label="Dots" value={value.dotColor}
      onChange={(hex) => onChange({ ...value, dotColor: hex })} />
  </div>

  <!-- Background color picker (CUST-02) -->
  <div data-testid="color-bg">
    <ColorPicker label="Background" value={value.bgColor}
      onChange={(hex) => onChange({ ...value, bgColor: hex })} />
  </div>

  <!-- Low-contrast warning (CUST-07) — non-blocking, inline -->
  {isLowContrast(effective_fg, value.bgColor) && (
    <div data-testid="low-contrast-warning" role="alert"
      className="mt-2 text-xs text-amber-700 bg-amber-50 border border-amber-200 rounded px-3 py-2">
      Low contrast — QR may not scan reliably
    </div>
  )}
  <!-- Note: "effective_fg" = gradientStop1 when gradient is enabled, dotColor when disabled -->

  <!-- Gradient controls (CUST-03) -->
  <div className="mt-4">
    <label className="flex items-center gap-2 cursor-pointer">
      <input type="checkbox" data-testid="gradient-toggle"
        checked={value.gradientEnabled}
        onChange={(e) => onChange({ ...value, gradientEnabled: e.target.checked })}
        className="w-4 h-4 accent-blue-600" />
      <span className="text-sm text-gray-700">Enable gradient</span>
    </label>

    {value.gradientEnabled && (
      <div className="mt-3 space-y-2 pl-6">
        <select data-testid="gradient-type"
          value={value.gradientType}
          onChange={(e) => onChange({ ...value, gradientType: e.target.value as "linear" | "radial" })}
          className="text-sm border border-gray-200 rounded px-2 py-1 focus:ring-2 focus:ring-blue-600 focus:outline-none">
          <option value="linear">Linear</option>
          <option value="radial">Radial</option>
        </select>

        <ColorPicker label="Color stop 1" value={value.gradientStop1}
          onChange={(hex) => onChange({ ...value, gradientStop1: hex })} />
        <ColorPicker label="Color stop 2" value={value.gradientStop2}
          onChange={(hex) => onChange({ ...value, gradientStop2: hex })} />
      </div>
    )}
  </div>
</section>
```

**ColorPicker sub-component** (internal, not exported — co-located in the same file):
```typescript
function ColorPicker({ label, value, onChange }: {
  label: string; value: string; onChange: (hex: string) => void;
}) {
  return (
    <label className="flex items-center gap-2">
      <span className="text-sm text-gray-700 w-24">{label}</span>
      <input type="color" value={value} onChange={(e) => onChange(e.target.value)}
        className="w-8 h-8 rounded cursor-pointer border border-gray-200" />
      <input type="text" value={value}
        onChange={(e) => {
          if (/^#[0-9a-fA-F]{6}$/.test(e.target.value)) onChange(e.target.value);
        }}
        maxLength={7}
        className="w-24 font-mono text-sm border border-gray-200 rounded px-2 py-1
                   focus:ring-2 focus:ring-blue-600 focus:outline-none" />
    </label>
  );
}
```

- The warning banner uses `role="alert"` for accessibility
- No `useDebounce` in this component — debouncing happens in QRGeneratorIsland in Plan 05
